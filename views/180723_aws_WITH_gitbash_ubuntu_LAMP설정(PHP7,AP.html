180723_aws_WITH_gitbash_ubuntu_LAMP설정(PHP7,APACHE2,MYSQL)_for_wordPress

  • 가끔씩 update와 upgrade(or dist-upgrade) 를 호출한다.

gitbash 실행
  1. ssh -i "개인pem경로" "계정이름"@"도메인(or publicip)" 
    - git bash 터미널로 시큐어셀(Secure Shell,[SSH] - 네트워크 상 다른 컴퓨터에 로그인하거나 원격 시스템에서 명령 실행 및 파일 복사)프로토콜로 연결한다. 이때, 미리 지정하여 프라이빗 키로 만든 아마존의 키페어(확장자 pem, 2048비트 ssh-2 RSA 방식)의 경로를 적어주고, [계정이름]을 입력 후, 서버 도메인을 입력하여 연결한다.
  2. sudo apt-get install apache2
    - superuser 권한으로 패키지관리 명령어 도구([APT], Advanced Packaging Tool)를 활용해 아파치 서버를 설치한다.
  3. sudo apt-get install php
    - 서버 언어 php를 설치한다.(여기서 주의사항은 ubuntu 16버전부터 php5를 지원하지 않는다. 때문에 php7 이상을 사용하는데, 이때문문에 라이브러리 모듈을 설치해준다/호환성)
    - 에러메시지 내용 : Package 'php5' has no installation candidate
    - php version 확인 : php -v
  4. sudo apt-get install libapache2-mod-php7.0
    - 위에서 언급한 모듈 설치
  5. a2query -m php7.0
    - php의 버전 확인 및 사용가능 여부 확인 >> php7.0 (enabled by maintainer script)
  6. sudo /etc/init.d/apache2 restart
    - 아파치웹서버 리부트(변경적용)
  7. cd /var/www/html
    - 실제적으로 view를 보여주는 웹상에서의 최상위 루트인 [DocumentRoot]이다. 파일 생성을 위해 이동.
  8. sudo vi info.php
    - 에디터를 통해서 info.php를 만들어준다.(다른 에디터를 사용하거나, ftp서버를 통해 파일을 추가할수도 있다.)
    - 여기서 경로 실수를 통해 파일이 다른 위치에 만들어졌다면 [mv] 명령어를 사용한다. : mv "이동 및 변경하고자하는 원본파일 이름" "이동을 원하는 경로 및 바꾸려고하는 파일 이름" : sudo mv info.php html/
  9. 도메인/info.php : php정보확인
  10. sudo apt-get install mysql-server
    - mysql을 설치한다.(중간에 root 암호를 지정한다)
  11. sudo apt-get install php-mcrypt php-mysql php-cli php-common
    - 사실상 이부분은 필수 모듈 설치. php-mysql은 필수.
  12. apache2와 mysql을 재시작한다. (참고 :6번 /etc/init.d/apache2 restart)
  13. sudo netstat -nlp
    - 네트워크의 [포트상태]가 보여진다. 이때, 3306,22,80 포트가 열려있다면 ok. : [LISTENING / PORT-FORWARDING]
  14. *optional* sudo apt-get install phpmyadmin
    - 웹상에서 데이터베이스를 손쉽게 관리하기 위해서 설치하는 dbms web GUI를 설치한다(선택적)
    - apache2? mysql 만 'space bar'로 선택하여 enter.

여기까지 Ubuntu LAMP 초기 설정 및 웹 db관리툴 생성.
