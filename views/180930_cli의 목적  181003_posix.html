file과 directory
정리의 효율성 증대. 복잡성 감소
file : 개개인 
directory : 같은 목적의 다수
명령어
pwd : print working directory
/ : root directory(절대경로)
.. & ../ : 현재폴더에서 위로(상대경로)
./ : 현재위치에서(상대경로)
cd : change directory
ls : list 호출
ls -l : 자세히
ls -a : 제외없이(숨긴파일형식도 가져온다)
mv PATH & NAME PATH2 & NAME2 : move and update
rm : remove
rm -r : 안전한 삭제(안전핀)
posix 계열의 운영체제에선 파일이름 앞에 .000.txt 와 같이
앞에 .이 들어가면 숨긴파일로 인식한다.

상대경로? 상대적으로 변하는 경로. 변화한다
절대경로? 절대적인 고정 경로. 변하지 않는다
GUI 는 과정을, 행동을 배우면 너무 쉽다. 트리구조를 구성한다.
CLI 는 언어를 배우는 것과 같다. 마인드맵, 웹구조를 구성한다.

시간의 순서에 따라 일을 진행할 수 있도록 CLI는 명령한다.
GUI는 다양한 가능성을 배제하고 자주 사용되는 명령을 그 순간 실행한다.

공장으로 치면,
과자를 포장하는 거대한 기계가 있다.
그 기계엔 ON / OFF / EMERGENCY 총 3개의 버튼이 있다.
이것은  GUI다.
ON 버튼을 누르는 순간, 자동적으로 5초에 한번씩 포장지를 자르고, 그후 벨트는 움직이면서 과자를 담고, 포장을 한다.
이 일은 지속적인 반복적인 일이다.
이 명령은 CLI로 반복적으로 적용된다.

GUI는 이처럼 우리의 행동에너지가 소비되는 대신, 시각적으로 편리함을 주고, 전문성이 없더라도 누구나 한번에 이해할 수 있는 구조로 갖춰진다.(UI/UX 디자인이 중요한 이유이기도 하다)
위와는 반대로 CLI의 경우, 우리가 영어를 배워 사용하듯이 배움이 있어야 활용할 수 있다. 그만큼 전문성이 필요하다. 그리고 많이 익히고 배울수록, 다양한 환경에서 적용이 가능하다.

한줄로 명령을 계속 입력할때.
; : 각 명령을 구분짓는  command seperator
&& : and operator "앞선 명령이 실행되고 난 후"

&&를 사용한다면 좀 더 정교한 명령을 내릴 수 있다.
shell script : computer automatic system을 구성할 수 있다
