  1. 사운드 압축 및 복원 기술
    1. FLAC - Free Lossless Audio Codec. 무손실 압축 포맷
    2. AIFF - Audio Interchange File Format. 비압축 무손실 포맷. Mac OS에서 표준으로 사용
    3. H.264(MPEG-4/AVC, MPEG-4 Part 10) - 고선명 비디오의 녹화,압축,배포를 위한 기장 일반적인 포맷. 매우 높은 데이터 압축 효율. - 비디오코덱
    4. WAV/WAVE - 무압축 방식. 아날로그 형태의 소리를 데이터 손실 없이 그대로 디지털 형태로 만들어 뛰어난 음질을 보여줌.

  1. 그래픽 데이터 표현 방식
    1. 비트맵 방식
      1. PIXEL 이라고 하는 여러 개의 점들로 이미지를 표현하는 방식.
      2. 벡터 방식의 이미지를 저장했을 때보다 메모리를 많이 차지.
      3. 그림을 확대하면 깨져보이는 현상 발생.(엘리어싱 현상)
      4. 고해상도 이미지를 표현하는데 적합.\
    2. 백터 방식
      1. 점과 점을 연결하는 직선이나 곡선을 이용하여 이미지 표현.
      2. 그림을 확대 OR 축소할 때 화질 손상이 거의 없음.
      3. 고해상도 이미지를 표현하기에는 부적합.

  1. DNS가 가지고 있는 특정 도메인의 IP Address 를 검색해 주는 서비스 : Nslookup
    1. Gopher(고퍼) : 메뉴 방식을 이용해 손쉽게 정보 검색을 할 수 있도록 하는 서비스
    2. Archie(아키) : FTP사이트에 있는 FTP서버와 파일정보를 DB에 저장해 두었다가  필요할 때 제공함으로써 정보를 쉽게 검색할 수 있도록 하는 서비스
    3. IRC : 인터넷 상에서 채팅을 할 수 있게 하는 서비스

  1. OSI 7계층 사용 장비.
    1. 물리계층(1계층) : Hub(허브), Repeater(리피터)
    2. 데이터 링크 계층(2계층) : (스위치),(랜카드),(브릿지)
    3. 네트워크 계층(3계층) : Router(라우터), L3 스위치
    4. 전송 계층(4계층) : Gateway(게이트웨이), L4 스위치

  1. 전자우편 프로토콜 POP3(Post Office Protocol 3) :메일 서버에 도착한 메일을 사용자 컴퓨터로 가져와 관리. 전자우편 수신용 프로토콜
  2. SMTP 프로토콜 : 사용자의 컴퓨터에서 작성한 메일을 다른 사람의 계정이 있는 곳으로 전송해주는 역할.
  3. MIME 프로토콜 : 웹 브라우저가 지원하는 각종 멀티미디어 파일의 내용을 확인한 후 실행해준다.
  4. IMAP 프로토콜 : 메일을 패킷으로 나누어 패킷주소를 해석하고 경로를 결정하여 메일 서버로 보내준다.

  1. 유니코드(Unicode)
    1. 세계 각국의 언어를 통일된 방법으로 표현할 수 있게 제안된 국제 코드 규약
    2. 한글은 조합형, 완성형, 옛글자 모두를 표현
    3. 65.536자(2의 16승)의 글자를 코드화
    4. 7 bit 인 ASCII코드를 16 bit 로 확장하여 전 세계의 모든 문자를 표현

  1. 셰어웨어(ShareWare) : 사용기간과 일부 기능을 제한하여 정식 제품의 구입을 유도하기 위한 프로그램.

  1. Cache(캐시) 메모리?
    1. 주로 SRAM 에 사용하며 주기억장치에 비해 용량이 적다.
    2. 캐시 적중률이 높을 수록 컴퓨터의 퍼포먼스가 향상.
    3. CPU와 주기억장치 사이에서 처리속도를 향상시키기 위한 일종의 버퍼 메모리 역할.

  1. 무정전 전원 공급장치(UPS) 는 과전류, 과전압을 보호할 때 쓰는것이 아닌 정전같이 전원이 갑자기 내려갔을 때 컴퓨터의 자료를 보호하기 위해 잠시동안 켜져있게 하는 장치.

  1. 중앙처리장치와 입출력장치 사이의 속도차이로 인한 문제를 해결하기 위한 장치?
    1. Channel (채널) 
      1. 입출력장치와 중앙처리장치 중간에 위치해 두 장치를 연결하는 중개 역할 담당.
      2. 입출력 장치의 명령 실행 지시 및 지시된 명령의 실행 상황을 점검.
      3. CPU 의 명령을 받고 입출력 조작을 개시하면 CPU와는 독립적으로 조작.

  1. 바이러스
    1. 연결형 바이러스 - 프로그램을 직접 감염시키지 않고 디렉토리 영역에 저장된 프로그램의 시작 위치를 바이러스의 시작 위치로 변경하여 프로그램 실행시 원래 프로그램 대신에 바이러스가 먼저 실행.
    2. 기생형 바이러스 - 원래 프로그램에 손상을 주지 않으면서 앞이나 뒤에서 공존하는 바이러스.
    3. 산란형 바이러스 - COM 파일을 생성하여 그 파일에 바이러스 프로그램을 넣어두는 바이러스. EXE 파일보다 COM파일이 먼저 실행되므로 바이러스에 감염된 것 같은 효과를 나타낸다.
    4. 겹쳐쓰기형 바이러스 - 원래의 프로그램의 앞부분에 겹쳐져서 바이러스 프로그램이 위치하는 경우로 원래의 파일 크기가 변화하지 않는다. 파일 실행 시 바이러스 프로그램 실행 및 원래 프로그램 파괴.

  1. 멀티미디어 동영상에 관련된 설명.
    1. 국제 표준화 단체 MPEG 에서는 다양한 규격의 압축 포맷과 부가 표준을 만듬.
    2. 비디오 스트리밍은 인터넷에서 영상파일을 다운로드 하면서 실시간 재생하는 기법.
    3. 퀵타임 MOV (Quick Time MOV) : '애플'사에서 개발한 동영상 압축 기술, JPEG 압축 방식을 사용.

  1. 머티미디어 그래픽 랜더링(Rendering) 기법
    1. 디더링 : 제한된 색상을 조합하여 새로운 색을 만드는 기술.
    2. 모핑 : 2개의 이미지를 부드럽게 연결하여 변환하는 기술.
    3. 안티앨리어싱 : 그림의 경계선을 부드럽게 처리해주는 필터링 기술.

  1. 프로토콜의 기능
    1. 연결제어 : 연결 설정 - 데이터 전송 - 연결 해제
    2. 흐름제어 : 통신망에 흐르는 패킷 수를 조정하여 시스템 전체의 안정성 유지.
    3. 오류제어 : 전송중에 발생한 오류를 검출하고 복원.
    4. 동기화(Synchronization) : 송/수신 축이 같은 상태를 유지하도록 타이밍을 맞추는 기능.
